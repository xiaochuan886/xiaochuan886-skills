# 文章检查清单

## 内容质量

- [ ] 标题有判断，但不夸张
- [ ] 篇幅在 1500-3000 字
- [ ] 结构完整：问题切入、背景、判断、案例/路径、启发、软 CTA
- [ ] 语气清晰，不是空泛励志文

## 引用规范

- [ ] 政策或法税主题至少有 1 个官方来源
- [ ] 案例主题至少有 1 个可追溯来源
- [ ] 参考资料列在文末
- [ ] `article.md` 中不包含 URL

## 配图检查

- [ ] 封面图已生成并存在
- [ ] 正文至少有 2 张配图
- [ ] 图片说明文字完整
- [ ] 图注单独成行，采用 `*图注*` 形式

## 合规检查

- [ ] 无收益保证
- [ ] 无“加入社群就能变现/接单/拿结果”表述
- [ ] 无强营销和焦虑制造
- [ ] 无个性化法税结论
- [ ] 涉及政策、注册、税务时标注了日期和地区

## 派生稿检查

- [ ] `xhs.md` 已生成
- [ ] 小红书版本保留同一核心判断
- [ ] 小红书版本仍然是软 CTA

## HTML 兼容检查

- [ ] frontmatter 已补 `summary`
- [ ] 标题下有 1 条摘要引用块
- [ ] 关键对照信息优先用 Markdown 表格或列表表达
- [ ] 没有大段原始 HTML 影响转换稳定性
