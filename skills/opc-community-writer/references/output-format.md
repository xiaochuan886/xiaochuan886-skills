# 信息检索输出格式

用于 `research.md` 的标准输出结构。

## 标准模板

```markdown
# Research Notes

## Topic
- Working title:
- Column:
- Date:

## 政策法规
- Source type: 官方政策
  发布机构:
  发布时间:
  适用地区:
  适用对象:
  核心结论:
  URL:
  对社群读者的价值:
  风险提醒:
  可转化角度:

## 办事路径
- Source type: 官方办事指南
  发布机构:
  发布时间:
  适用地区:
  适用对象:
  办理路径:
  关键材料/步骤:
  URL:
  对社群读者的价值:
  风险提醒:
  可转化角度:

## 法律边界
- Source type: 法院解读 / 法治文章 / 监管口径
  发布机构:
  发布时间:
  适用地区:
  适用对象:
  核心边界:
  URL:
  对社群读者的价值:
  风险提醒:
  可转化角度:

## 创业案例
- Source type: 创业案例 / 媒体报道
  案例主体:
  发布时间:
  地区:
  案例摘要:
  可验证事实:
  URL:
  对社群读者的价值:
  风险提醒:
  可转化角度:

## 平台内容观察
- Platform:
  Account:
  Title:
  Date:
  Angle:
  Notes:
  URL:
```

## 检索优先级

| 优先级 | 来源类型 | 说明 |
|--------|----------|------|
| 1 | 官方政策与办事指南 | 用于政策、注册、补贴、办事路径 |
| 2 | 法院/法治/监管口径 | 用于公司法、责任边界、风险提醒 |
| 3 | 官方或准官方项目 | 用于园区、社区、扶持项目、活动 |
| 4 | 权威媒体与案例 | 用于案例拆解、趋势补充 |
| 5 | 平台内容观察 | 用于竞品视角和标题灵感 |

## 硬规则

- 政策或法税主题至少要有 1 条官方来源
- 案例主题至少要有 1 条可追溯案例来源
- 每条都要有 `发布时间`、`适用地区/地区` 和 `URL`
- `research.md` 可以保留 URL，`article.md` 和 `xhs.md` 不能保留 URL
